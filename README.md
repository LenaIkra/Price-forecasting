# Прогнозирование цен потребительского ритейла с использованием методов машинного обучения

Репозиторий содержит программную реализацию выпускной квалификационной работы по теме:

**«Прогнозирование цен потребительского ритейла по тестовой выборке с использованием методов машинного обучения»**

Проект посвящен исследованию методов прогнозирования цен в потребительском ритейле на основе временных рядов, данных о продажах, ценах и календарных событиях.

---

## Цель проекта

Цель работы — разработка модели прогнозирования цен в потребительском ритейле на основе методов машинного обучения и анализа временных рядов.

## Что реализовано

В проекте реализован полный pipeline прогнозирования цен:

- загрузка исходных таблиц M5 Forecasting Dataset;
- объединение данных о продажах, ценах и календарных событиях;
- построение аналитического датасета;
- формирование лаговых, календарных, скользящих и иерархических признаков;
- обучение моделей прогнозирования;
- подбор гиперпараметров с использованием Optuna;
- сохранение моделей, прогнозов, метрик и параметров;
- статистическая проверка различий между лучшими моделями;
- проверка модели на независимой подвыборке временных рядов;
- расчет потенциального экономического эффекта.

---

## Используемый набор данных

В работе использовался набор данных [**M5 Forecasting Dataset**](https://www.kaggle.com/competitions/m5-forecasting-accuracy/data), содержащий информацию о продажах товаров, ценах и календарных событиях сети Walmart за период 2011–2016 гг.

В рамках исследования данные были преобразованы в аналитический панельный датасет, включающий:

- данные о продажах товаров;
- цены товаров;
- календарные события;
- признаки магазинов и товарных категорий;
- лаговые признаки;
- признаки скользящих средних.

Исходные данные не включены в репозиторий из-за большого объема и должны быть загружены пользователем самостоятельно из Kaggle.

## Разведочный анализ данных

Перед построением моделей был выполнен разведочный анализ данных. В рамках EDA анализировались структура исходных таблиц, распределение продаж и цен, наличие пропусков, сезонность, календарные события, разреженность временных рядов и особенности товарной иерархии.

Материалы разведочного анализа доступны в ноутбуке: [EDA](https://colab.research.google.com/drive/1tdDVElVk9w_vOIv2fqpCvG4Il-NfwluY?usp=sharing)

## Исследуемые модели

В рамках вычислительного эксперимента были реализованы и сравнены следующие модели:

| Модель | Краткое описание |
|---|---|
| SARIMA | Классическая статистическая модель временных рядов |
| Gradient Boosting | Базовая модель градиентного бустинга на табличных признаках |
| M5-style Boosting | Модель LightGBM с логарифмированием целевой переменной и иерархическими признаками |
| LSTM + Embeddings | Рекуррентная нейронная сеть с embedding-представлениями категориальных признаков |
| Hybrid Boosting + M5-style | Взвешенный ансамбль прогнозов Gradient Boosting и M5-style Boosting |
---

## Метрики качества

Для оценки качества прогнозирования использовались следующие метрики:

- **RMSE** — корень из среднеквадратичной ошибки;
- **MAE** — средняя абсолютная ошибка;
- **WAPE** — взвешенная абсолютная процентная ошибка.

Основной метрикой сравнения моделей была выбрана **RMSE**, поскольку она сильнее штрафует крупные ошибки прогнозирования.

---

## Итоговые результаты

Финальное сравнение моделей выполнялось на единой подвыборке из 5000 наиболее полных временных рядов.

| Модель | RMSE | MAE | WAPE |
|---|---:|---:|---:|
| Hybrid Boosting + M5-style | 0.02255 | 0.00374 | 0.00088 |
| M5-style Boosting | 0.02294 | 0.00373 | 0.00088 |
| Gradient Boosting | 0.05302 | 0.01311 | 0.00308 |
| LSTM + Embeddings | 1.45699 | 1.26032 | 0.29586 |
| SARIMA | 5.44169e+62 | 2.27895e+60 | 5.34727e+59 |

Лучшее значение основной метрики RMSE показала гибридная модель Hybrid Boosting + M5-style. При этом дополнительная статистическая проверка показала, что различие между Hybrid Boosting + M5-style и M5-style Boosting не является статистически значимым. Поэтому в качестве основной самостоятельной модели для интерпретации результатов рассматривается M5-style Boosting, а гибридная модель используется как дополнительный ансамблевый вариант.

---

## Проверка статистической значимости

Для проверки различий между лучшими моделями была выполнена bootstrap-оценка доверительных интервалов разности метрик.

Проверялись различия между моделями:

- Hybrid Boosting + M5-style и M5-style Boosting;
- Gradient Boosting и M5-style Boosting;
- Gradient Boosting и Hybrid Boosting + M5-style.

Результаты проверки показали:

- различие между Hybrid Boosting + M5-style и M5-style Boosting статистически незначимо;
- M5-style Boosting статистически значимо лучше базовой модели Gradient Boosting;
- Hybrid Boosting + M5-style статистически значимо лучше базовой модели Gradient Boosting.

Результаты проверки сохраняются в файл:

```text
reports/final_run/significance_tests/model_significance_tests.csv
```

---
## Независимая валидация

Для дополнительной проверки устойчивости модели была проведена оценка M5-style Boosting на независимой подвыборке временных рядов, не входивших в основную подвыборку top-5000.

Результаты независимой проверки:

| Выборка | RMSE | MAE | WAPE |
|---|---:|---:|---:|
| Основная тестовая выборка top-5000 | 0.02294 | 0.00373 | 0.00088 |
| Независимая подвыборка | 0.02939 | 0.00506 | 0.00120 |

При переходе к независимой подвыборке качество прогнозирования снизилось, однако модель сохранила работоспособность и не показала численной нестабильности.

Результаты сохраняются в файлы:

```text
reports/final_run/independent_validation/m5_style_metrics_with_ci.csv
reports/final_run/independent_validation/m5_style_metrics_with_ci.json
```

## Основные модули проекта

Проект построен по модульному принципу: загрузка данных, подготовка признаков, обучение моделей, оценка качества и расчет прикладных показателей вынесены в отдельные файлы. Это упрощает повторный запуск экспериментов, доработку отдельных моделей и воспроизведение результатов.

| Модуль | Назначение |
|---|---|
| src/config.py | хранение основных параметров проекта: путей, настроек данных, признаков и обучения |
| src/data_loader.py | загрузка исходных таблиц M5 Forecasting Dataset: продаж, цен и календарных событий |
| src/dataset_builder.py | объединение исходных таблиц и построение итогового аналитического датасета |
| src/splits.py | разделение данных на обучающую и тестовую выборки с сохранением временного порядка |
| src/final_pipeline.py | основной pipeline вычислительного эксперимента: запуск моделей, кеширование результатов, сохранение метрик |
| src/final_results_manager.py | формирование итоговых таблиц сравнения моделей |
| src/train_sarima.py | обучение и оценка статистической модели SARIMA |
| src/train_boosting.py | обучение базовой модели Gradient Boosting |
| src/train_m5_style.py | обучение модели M5-style Boosting с учетом иерархических признаков и логарифмирования целевой переменной |
| src/train_lstm.py | обучение LSTM-модели с embedding-представлениями категориальных признаков |
| src/train_hybrid_boosting_m5.py | построение гибридной модели на основе прогнозов Gradient Boosting и M5-style Boosting |
| src/model_significance_test.py | проверка статистической значимости различий между лучшими моделями |
| src/independent_validation.py | проверка модели M5-style Boosting на независимой подвыборке временных рядов |
| src/economic_effect.py | расчет потенциального экономического эффекта на основе ошибок прогноза цены и объемов продаж |
| main_final.py | основной файл запуска итогового эксперимента |


Такая структура позволяет запускать как полный вычислительный эксперимент, так и отдельные этапы проекта: независимую проверку модели, расчет экономического эффекта или анализ статистической значимости различий между моделями.

## Запуск основного pipeline

Основной вычислительный эксперимент запускается командой:

```bash

python3 -u main_final.py

```

Параметр `-u` используется для вывода логов в режиме реального времени.

Pipeline выполняет следующие этапы:

- загрузка исходных данных;

- построение аналитического датасета;

- разделение данных на обучающую и тестовую выборки;

- формирование подвыборки top-5000 временных рядов;

- обучение моделей;

- сохранение прогнозов, метрик и параметров;

- формирование итоговой таблицы сравнения моделей.
